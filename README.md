## ⚙️ Como Funciona

Este portfólio **OctoMind** é automatizado através de:

1. **Script Node.js**: Coleta dados dos repositórios via API do GitHub
2. **GitHub Actions**: Executa o script semanalmente e quando necessário
3. **GitHub Pages**: Publica o resultado como um site estático

## 🛠️ Tecnologias

- Node.js
- GitHub Actions
- GitHub Pages
- GitHub API

## 🧠 Desafios e Aprendizados

* **Principais desafios técnicos enfrentados:**
  * Equilibrar a performance do sistema com a simplicidade de uso, especialmente ao lidar com integrações de backend, como autenticação, agendamento inteligente e filas de espera.
  * Definir uma arquitetura escalável desde o início, pensando na manutenção e futuras expansões — um desafio comum em MVPs rápidos.
  * Lidar com questões de conectividade e disponibilidade em regiões mais remotas, onde fatores externos influenciam diretamente na entrega do serviço (como vivenciado em projetos ligados à M2 Ultra Fibra e à Bonito Ecoexpedições).

* **Aprendizados significativos durante o desenvolvimento:**
  * Aprimorei minha capacidade de transformar necessidades locais em soluções digitais viáveis, conectando conhecimento técnico à realidade de pequenos prestadores e empresas regionais.
  * Aprofundei meu domínio em infraestrutura de rede, incluindo firewall com Mikrotik, VLANs em Huawei e integração com Proxmox e OpenMediaVault, aplicando isso diretamente em projetos com foco em monitoramento (como o uso do Uptime Kuma).
  * Aprendi a priorizar a experiência do usuário desde o primeiro rascunho, integrando interfaces limpas, modernas e acessíveis, mesmo em soluções minimalistas.

* **Tecnologias e conceitos explorados:**
  * Utilização de GitHub Actions para automação de CI/CD, integração com bancos PostgreSQL e Redis, além de estilização com TailwindCSS e interface dinâmica com React e Next.js.
  * Criação de sistemas robustos para agendamento, gerenciamento de frota e controle financeiro local — sempre com foco na autonomia do usuário final e fácil manutenção.
  * Aprendizado e aplicação de boas práticas com Docker, controle de versões, deploy contínuo e documentação clara para futuras colaborações.

* **Como os desafios foram superados:**
  * Busquei soluções inteligentes e bem documentadas, muitas vezes conciliando a documentação oficial com tutoriais específicos da comunidade.
  * Mantive o foco em entregas incrementais, testando com usuários reais — como motoristas, prestadores e administradores locais — para validar funcionalidades.
  * Estabeleci uma rotina de melhorias constantes, tratando feedbacks como bússolas e adaptando a stack de acordo com a maturidade do projeto.

> 💡 **Nota pessoal:** Encarar a criação de soluções locais como oportunidades de inovação é o que mais me motiva. Cada obstáculo vencido é também uma chance de gerar impacto positivo na vida real.
